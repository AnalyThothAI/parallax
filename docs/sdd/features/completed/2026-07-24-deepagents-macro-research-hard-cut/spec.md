## Background

The retired Macro product had two derived judgment systems over the same
`macro_observations` facts
(`docs/sdd/features/completed/2026-07-23-daily-macro-spy-judgment/spec.md:74`):

1. a deterministic six-page `macro_decision_v2` projection whose overview
   contained eight deterministic risk-exposure lanes
   (`docs/sdd/features/completed/2026-07-23-daily-macro-spy-judgment/spec.md:74`); and
2. a Daily SPY 5D/20D model lane whose EvidencePack, language, readiness,
   degradation, `no_call`, review order, and publication eligibility are
   predominantly decided by Parallax code
   (`docs/references/deepagents-core-design-audit-2026-07-24.md:66`).

The installed DeepAgents runtime is real, but the current integration reduces it
to a fixed Analyst submission followed by a forced Reviewer task. The product
therefore presents deterministic placeholders as professional judgment, blocks
model capability behind semantic gates, and can publish English prose into a
Chinese product
(`docs/references/deepagents-core-design-audit-2026-07-24.md:67`).

## Problem

Macro is nearly unusable because the visible product is generated by rule files
and defensive gates rather than by an agent that can inspect the evidence,
decide what matters, pursue counterevidence, delegate research, and explain its
judgment in Chinese. Keeping a generic Parallax LLM gateway alongside a special
Macro DeepAgents adapter also creates two incompatible model-execution
architectures.

## Goals

- G1. Replace all code-authored Macro judgment with one DeepAgents-authored
  Chinese research artifact per completed U.S. regular session.
- G2. Give the agent native ownership of planning, evidence exploration,
  subagent delegation, counterevidence, review, and final semantic output.
- G3. Keep Parallax as a deep evidence/runtime adapter: frozen cutoff, typed
  read tools, durable run lifecycle, model composition, referential integrity,
  and atomic persistence.
- G4. Expose one coherent `/macro` decision surface instead of six rule pages
  plus an unrelated Daily card.
- G5. Delete the dormant generic LLM gateway and every retired compatibility
  path.

## Non-goals

- N1. No deterministic direction, confidence, score, probability, regime,
  shock, risk-lane, readiness, coverage, language, or `no_call` engine.
- N2. No SPY-only 5D/20D forecast schema, trade instruction, position sizing,
  entry, stop, target, or portfolio action contract.
- N3. No request-time provider or model execution.
- N4. No new Parallax workflow DSL, generic agent platform, model gateway,
  compatibility wrapper, dual-write, or fallback renderer.
- N5. No deletion of `macro_observations`, sync windows/runs, source identity,
  raw provenance, or provider adapters.
- N6. No claim that a structured envelope or citation-integrity check is a
  semantic judgment gate.

## Interface contracts

### Deep module

`CompletedSessionMacro.run(session_date: date | None = None)` is the single
write-side product interface and `CompletedSessionMacro.read(...)` is its
persisted-only read seam. The worker invokes zero-argument `run()` for normal
catch-up; an explicit date is reserved for backfill. Its caller does not choose
a program, tool order, research sections, risk taxonomy, readiness state, or
review sequence.
Internally it:

1. opens a frozen evidence scope at the session close and run seal time;
2. gives one DeepAgent native todo planning, checkpoint filesystem, shared
   execute workspace, real `execute`, scoped evidence/news/history tools, and
   declarative specialist subagents;
3. lets the Agent decide what to inspect, delegate, challenge, and include;
4. accepts one structured Chinese research artifact;
5. mechanically verifies session identity and citation closure; and
6. atomically publishes the immutable artifact and completes the run.

The worker owns only completed-session scheduling, leases, retries, and calling
this interface. Model/tool I/O occurs outside database write transactions.
Search tools expose compact offset pages without a total-depth ceiling, while
exact-reference reads preserve full evidence and rely on native checkpoint
filesystem offload for large results. Parallax does not turn a legal large
result into a semantic or runtime failure.

### Research artifact

The persisted artifact has a stable mechanical envelope and agent-owned
contents:

- session date, market cutoff, title, and executive summary;
- one authoritative dynamic ordered list of agent-named Markdown sections;
- an agent-authored list of material evidence gaps and open questions;
- citations containing stable IDs, source labels, observation/publication
  times, URLs when available, and material-fact/news lineage;
- reviewer notes and run audit metadata.

The envelope does not prescribe asset lanes, directions, confidence,
readiness, fixed sections, forecast horizons, or a deterministic conclusion.
Production code does not reject an artifact for its language, stance,
completeness judgment, or professional opinion. Parsing and citation closure
remain mandatory because they are storage integrity, not investment judgment.

### Public read

`GET /api/macro/research` returns the latest publication by default and accepts
an explicit completed `session_date`. It returns `current`, `historical`,
`generating`, `failed`, or `missing` state from persisted rows only. The
endpoint never calls the model, a provider, or a projection builder.

`/macro` is the only Macro browser route. It renders the persisted Chinese
artifact's authoritative ordered Markdown sections, evidence gaps, citations,
session/cutoff, and generation/error state. A flat Markdown export is
mechanically derived from those sections rather than authored as a second
narrative. The five child Macro routes and all deterministic page/series/Daily
endpoints are removed.

## Acceptance criteria

- AC1. WHEN Macro material evidence is ingested THEN `macro_observations`, `macro_sync_windows`, and `macro_sync_runs` SHALL remain the sole Macro facts and sync attempt state.
- AC2. WHEN a completed session is due THEN one bounded `macro_research` worker SHALL call the zero-argument `CompletedSessionMacro.run()` interface, with model and tool I/O outside a database write transaction.
- AC3. WHEN the research graph runs THEN the parent DeepAgent SHALL have native todo planning, checkpoint filesystem scratch and large-result offload, shared `/workspace/`, real `execute`, dynamic `task` delegation, scoped evidence/news/history tools, and structured final output without a Parallax-authored tool-order FSM or tool-exclusion layer.
- AC4. WHEN the Agent researches THEN it SHALL query facts through a frozen session cutoff, page compact searches, retain complete exact-read evidence through native checkpoint-filesystem offload, and decide its own evidence selection, gaps, section structure, counterevidence, subagent tasks, and final semantic conclusions.
- AC5. WHEN a structured result returns THEN Parallax SHALL validate only the mechanical envelope, exact session/cutoff identity, and citation closure before immutable publication without language, readiness, coverage, direction, confidence, or `no_call` gates.
- AC6. WHEN the same published session is replayed THEN publications SHALL cause zero model calls, zero serving writes, and reject mutation or deletion.
- AC7. WHEN `/api/macro/research` reads latest or explicit history THEN it SHALL return typed persisted state without provider or model calls.
- AC8. WHEN `/macro` renders THEN it SHALL show one coherent Chinese research document with agent-authored sections, gaps, citations, and audit context on desktop and mobile without horizontal overflow.
- AC9. WHEN source and database residuals are scanned THEN old `macro_decision_v2`, eight-lane rules, Daily SPY judgment, projection and judgment tables, workers, settings, endpoints, routes, and components SHALL be absent.
- AC10. WHEN model-execution residuals are scanned THEN the dormant generic execution gateway, schema strategy, usage wrapper, agent capabilities, execution and hash platform modules, and direct LiteLLM dependency SHALL be absent.
- AC11. WHEN a real configured provider run is performed THEN a completed session SHALL publish a Chinese artifact with closed persisted citations, planning and subagent audit, and a passing independent blind semantic review.
- AC12. WHEN completion is claimed THEN non-empty PostgreSQL migration, selected backend and frontend gates, generated contracts, browser inspection, docs, residual scans, and SDD verify gates SHALL have direct successful evidence.
