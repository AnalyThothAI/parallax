Why This Variant Exists

product rule
Event types deserve modules

A hack item needs attack path and assets. A regulatory item needs authority and instrument mapping.

Do not overfit one layout

The page shell stays consistent, but the middle module should adapt to event_type.

Product recommendation: combine A as the default item route, B as an optional dense mode for advanced users, and D as an event-type module inside A for hacks/security events. C is useful for shareable or analyst-note mode, but should not be the default for high-frequency trading workflows.
