Obsidian Desk v2

把实时噪音整理成可判断的案卷台

这版设计语言把 Token Radar、Search Intel、Watchlist、Signal Pulse 统一成同一种交互语法： 先扫列表，再开案卷，最后看证据链和下一步动作。颜色不再是装饰，而是产品状态。

看 Radar 桌面 看二级页 看 Watchlist 看 Signal Pulse

Design Rules

01
一切都是 case fileToken、账号、主题、pulse 都有 brief、facts、evidence、decision。
02
颜色绑定语义Amber 是机会，Green 是健康，Blue 是信息，Red 是风险，Violet 是 agent。
03
列表负责扫，详情负责判列表只给足够判断是否点击的信息；深度解释统一进入右栏或二级页。
04
少用卡片，多用档案区块页面区块是整幅案卷，不堆叠浮动卡；重复实体才使用卡片。

Production Translation

落地拆分建议

这套视觉语言可以分三步落地，避免一次性重写所有页面。

1. Design tokens first

先替换全局 tokens：颜色、字体、线框、状态色、密度。生产代码可以保留现有组件结构，先统一气质。

  • 新增 semantic color tokens
  • 统一 pill / badge / segmented
  • 移除单一黄黑依赖

2. Radar case row

改 TokenRadarRow 信息模型，先把 official/community/narrative 进入 item 和 selected detail。

  • 复用 TokenProfileCard 数据
  • 行内摘要不做 agent 事实
  • Search Intel 成为主动作

3. Secondary pages

Search、Watchlist、Signal Pulse 共享 case file layout，降低用户学习成本。

  • Search: case brief + timeline
  • Watchlist: account file
  • Pulse: agent memo + fact ledger
